Elementos Semánticos en HTML

La semántica en HTML se refiere a utilizar etiquetas que describen el contenido de manera más clara. Esto ayuda a los motores de búsqueda y a los lectores de pantalla a entender mejor la estructura y el propósito del contenido en una página web. El uso adecuado de elementos semánticos también mejora la accesibilidad y la mantenibilidad del código.

Lista de Elementos Semánticos

  • <article>: Representa contenido autónomo que podría ser distribuido o reutilizado, como un artículo de una revista.

    Ejemplo: Un artículo de blog, una entrada de noticias o un comentario de usuario.

  • <aside>: Contenido relacionado que puede ser considerado como un "comentario" al contenido principal, como una barra lateral.

    Ejemplo: Un bloque de texto que incluye información adicional, como datos de autor o enlaces relacionados.

  • <details>: Un elemento que puede ser expandido o contraído para mostrar contenido adicional.

    Ejemplo: Un menú desplegable que revela información adicional sobre un tema específico.

  • <figcaption>: Proporciona un título o descripción para una figura (como imágenes o diagramas).

    Ejemplo: La leyenda de una imagen o gráfico que explica su contexto.

  • <figure>: Un contenedor para contenido ilustrativo, como imágenes, diagramas o gráficos.

    Ejemplo: Una ilustración acompañada de un texto descriptivo.

  • <footer>: Sección de pie de página que contiene información sobre el contenido, como el autor o derechos de autor.

    Ejemplo: Información de contacto, derechos de autor y enlaces de redes sociales.

  • <header>: Contenedor para encabezados que pueden incluir títulos, subtítulos o menús de navegación.

    Ejemplo: El encabezado de un artículo o el logo de un sitio web.

  • <main>: Representa el contenido principal del documento, excluyendo encabezados, pies de página y barras laterales.

    Ejemplo: El cuerpo principal de un artículo, que contiene la información más relevante.

  • <mark>: Resalta texto importante o relevante dentro de un contenido.

    Ejemplo: Palabras clave o frases que se desean destacar en un texto.

  • <nav>: Contenedor para enlaces de navegación que dirigen a otras partes de la misma página o a otras páginas.

    Ejemplo: Un menú de navegación que conecta a diferentes secciones de un sitio web.

  • <section>: Representa una sección temática de contenido, que generalmente incluye un encabezado.

    Ejemplo: Un capítulo en un libro o una sección de preguntas frecuentes.

  • <summary>: Proporciona un resumen para el contenido del elemento <details>.

    Ejemplo: Un resumen breve de lo que se encontrará al expandir un menú.

  • <time>: Representa un valor de tiempo específico o un rango de tiempo.

    Ejemplo: Fechas de eventos, como una conferencia o un lanzamiento de producto.

Ejemplos de Uso

Ejemplo de <article> y <aside>

En el siguiente ejemplo, el elemento <article> contiene contenido que puede ser leído de forma independiente, mientras que el <aside> proporciona información adicional relacionada.


<article>
    <header>
        <h2>Título del Artículo</h2>
    </header>
    <p>Contenido del artículo que es autónomo y puede ser distribuido.</p>
    <aside>
        <p>Este es un contenido adicional relacionado.</p>
    </aside>
</article>

Ejemplo de <figure> y <figcaption>

El siguiente ejemplo muestra cómo utilizar <figure> para incluir una imagen y su descripción correspondiente con <figcaption>.


<figure>
    <img src="ruta/a/tu-imagen.jpg" alt="Descripción de la imagen">
    <figcaption>Descripción de la imagen que acompaña a la figura.</figcaption>
</figure>

Ejemplo de <details> y <summary>

Este ejemplo muestra un menú desplegable donde el usuario puede hacer clic para ver más información.


<details>
    <summary>Más información</summary>
    <p>Este es el contenido adicional que aparece al expandir.</p>
</details>

Ejemplo de <nav>

Aquí se muestra un menú de navegación simple que permite a los usuarios acceder a diferentes secciones de la página.


<nav>
    <ul>
        <li><a href="#seccion1">Sección 1</a></li>
        <li><a href="#seccion2">Sección 2</a></li>
    </ul>
</nav>

Importancia de la Accesibilidad

El uso de elementos semánticos no solo mejora la estructura del contenido, sino que también facilita el acceso a la información para personas con discapacidades. Los lectores de pantalla y otras herramientas de accesibilidad utilizan las etiquetas semánticas para proporcionar una experiencia de usuario óptima.

Por ejemplo, el uso adecuado de etiquetas como <header>, <nav> y <main> permite que los usuarios naveguen de manera más eficiente por el contenido, identificando secciones importantes sin tener que recorrer todo el texto.

Esto también ayuda a los motores de búsqueda a clasificar mejor el contenido, ya que entienden el contexto de cada parte del documento, lo que puede mejorar el SEO (Optimización para Motores de Búsqueda).
